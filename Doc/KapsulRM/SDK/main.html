Kapsul SDK  
      Kapsul due to his open architecture, permits you to add custom plugins or completely
      change the user interface. Thanks to his Kore.dll, You can link whatever
      you want. This file contains all the classes needed for a 3D application.
      The Kapsul.exe is just a MDI container with some toolbars and a
      CView class used to rout user action. This class send actions either to
      the current plugins or the in-built functions. The most interresting part of this sdk
      is of course the ability to build two types of plugins (more to come later). You can totaly
      take control of the interface :
      • Add owner draw
      • Get all mouse and keyboard actions
      • know when user change plugin
      • know when user save desk
      • ...

      This is done via Windows-like messaging handling. you receive messages trough you Process
      Two more parameters wParam and lParam are, as the name says, parameters
      for the message. A retun value is ,as the two parameters above, a sometimes needed unsigned long

      You can build another type of plugin, more simple. I/O plugins are handled the same way modifier
      plugins are : Via messages

      Some mecanism permits you to have onwer-datas and owner-draw.
      All this particular features are described in this pages
