KM_DRAW

    Parameters :

    wParam : int State : State of the view to draw
    lParam : int WichOne : Zero-based indiced of the view

    Return values :
    (INT) value of the things to draw.

    Remarks :
    If the Editor configuration has four views visible, the message occurs four times at each
    redraw of the MDI window.

    State value is 0 four WireFrame drawing. View is mapped otherwise. Not so much state
    for the moment.
    WichOne indicates the indice of the view to be drawn (goes from 0 to 3) for all the view.
    With this INT, you can retrieves more information on the to-be-drawn view.

    (dk->ViewType[WichOne]==9) is TRUE if the to-be-drawn view is a camera. Is that
    case :
    dk->cam[WichOne] is a Pointer to a « Camera » object used to draw the view.

    The API you must use is OpenGL.

    The return value tells Kapsul what to draw after.

    It can be one or a combination of :

    PLUGIN_CAMERA
    PLUGIN_OBJECT
    PLUGIN_SHAPE
    PLUGIN_PARTICLE
    PLUGIN_SKELETTON
    PLUGIN_LIGHT
    PLUGIN_GRID
    PLUGIN_AXIS

    PLUGIN_ALL corresponds to PLUGIN_CAMERA|PLUGIN_OBJECT|PLUGIN_SHAPE|
    PLUGIN_PARTICLE|PLUGIN_SKELETTON|PLUGIN_LIGHT
    |PLUGIN_GRID|PLUGIN_AXIS PLUGIN_NONE corresponds to 0
